01 · Contexto

Pergunta de negócio

Combinações de armazém e modal de transporte criam padrões distintos de custo e desempenho logístico?

Uma empresa de e-commerce usa 5 blocos de armazém (A–F) e 3 modais (Flight, Road, Ship). A clusterização busca descobrir se essas combinações formam perfis operacionais naturais nos 10.999 pedidos do dataset.

Objetivo: agrupar pedidos em perfis logísticos homogêneos para identificar padrões e subsidiar decisões estratégicas — redirecionar produtos pesados para modais baratos, concentrar itens de alto valor em armazéns específicos etc.
02 · Dados

Variáveis utilizadas

Warehouse_block
Bloco do armazém de origem.
Encoding: A=0 B=1 C=2 D=3 F=4
Mode_of_Shipment
Modal de transporte.
Encoding: Flight=0 Road=1 Ship=2
Weight_in_gms
Peso do produto em gramas. Varia de ~1.000g a ~7.800g — sem outliers.
Cost_of_the_Product
Custo do produto. Maioria entre $150 e $250 — sem outliers.
Pré-processamento: encoding com df.map() + normalização com StandardScaler para colocar todas as variáveis na mesma escala antes de calcular distâncias euclidianas.

04 · Implementação

Código passo a passo

# Selecionar as 4 variáveis da operação logística cols_5 = ["Warehouse_block", "Mode_of_Shipment", "Weight_in_gms", "Cost_of_the_Product"] # Dados já normalizados com StandardScaler X5 = df_scaled[cols_5] # n_init=10 → testa 10 inicializações, escolhe a melhor (menor inércia) # random_state=42 → garante reprodutibilidade kmeans5 = KMeans(n_clusters=3, random_state=42, n_init=10) df["cluster_p5"] = kmeans5.fit_predict(X5) # Resultado real obtido: # cluster_p5 → 1: 5159 pedidos (47%) # → 0: 3728 pedidos (34%) # → 2: 2112 pedidos (19%) # Silhouette: 0.255
fit_predict(): fit() aprende os 3 centróides. predict() atribui cada pedido ao centróide mais próximo (distância euclidiana no espaço normalizado das 4 variáveis). Cada linha do dataframe recebe label 0, 1 ou 2.

Métricas que avaliam a qualidade do agrupamento sem usar rótulos externos.

Método do Cotovelo
Inércia por k — ponto laranja = k=3 escolhido
Davies-Bouldin Index
Menor = clusters mais distintos — ponto laranja = k=3
Silhouette Score por Cluster
Score médio estimado por cluster — quanto mais positivo, melhor alocado
Como interpretar o Silhouette

Para cada ponto: s = (b − a) / max(a, b) — onde a = distância média aos pontos do mesmo cluster e b = distância média ao cluster vizinho. Score 0.255 é comum para dados mistos (numérico + categórico codificado).

Silhouette 0.255 e DB 1.452 confirmam separação moderada — aceitável para dados mistos. Dica de prova: lembre da cadeia — pergunta → variáveis → encoding + normalização → K-Means validado pelo Cotovelo → Silhouette + DB internamente → Pureza + ARI externamente → interpretação dos 3 clusters. Cada etapa justifica a próxima.
